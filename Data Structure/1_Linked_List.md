# \# 연결리스트



- 연속적인 메모리 위치에서 저장되지 않는 선형 구조

  => 포인터를 사용하여 연결



##### 왜 배열 말고 Linked List를 사용?

> - 장점
>
> 1. 배열과 달리 크기가 고정되어 있지 않아서 동적 할당이 가능
> 2. 배열에 비해 새 요소 삽입/삭제 시 효율적
>
> 
>
> - 단점
>
> 1. 임의 액세스 불가, ***순차 접근*** 만이 가능
> 2. 포인터가 필요하므로 오버헤드가 발생





#### 구현법

- 맨 앞 노드 Head

- 맨 끝 노드는 NULL을 가리킴

- 특정 노드를 기준으로 삽입 / 삭제 구현

  ㄱ. 맨 앞에 노드 추가

  ㄴ. 특정 노드 다음에 추가

  ㄷ. 맨 뒤에 노드 추가
