## 운영체제란



- 일반적으로

  <u>하드웨어를 관리하고,</u> 

  <u>응용 프로그램과 하드웨어 사이에서 인터페이스 역할을 하며,</u>

  <u>시스템의 동작을 제어하는 시스템 소프트웨어</u>

  로 정의됨

  => ***시스템의 자원과 동작을 관리하는 SW***



#### \*\* 큰 틀에서의 OS 역할

1. 프로세스 관리
   - 프로세스, 스레드
   - 스케줄링
   - 동기화
   - IPC 통신
2. 저장장치 관리
   - 메모리 관리
   - 가상 메모리
   - 파일 시스템
3. 네트워킹
   - TCP / IP
   - 기타 프로토콜
4. 사용자 관리
   - 계정 관리
   - 접근 권한 관리
5. 디바이스 드라이버
   - 순차 접근 장치
   - 임의 접근 장치
   - 네트워크 장치



#### 1. 프로세스 관리

- OS에서 작동하는 응용 프로그램을 관리

  => CPU를 점유해야 할 프로세스를 선정, CPU에 할당, 프로세스 간 공유 자원 접근 및 통신 등의

  ​     작업을 진행하므로, ***CPU를 관리하는 것이라고도 볼 수 있음***



#### 2. 저장장치 관리

 	: 1차 저장장치인 메인 메모리, 2차 저장장치인 HDD, NAND 등을 관리



- 1차 저장장치

  - 프로세스에 할당하는 메모리 영역의 할당과 해제
  - 각 메모리 영역 간 침범 방지
  - 메인 메모리의 효율적 활용을 위한 가상 메모리 기능

- 2차 저장장치

  - 파일 형식의 데이터 저장

  - 파일 데이터 관리를 위한 파일 시스템을 OS에서 관리

    => 파일 시스템은 `FAT, NTFS, EXT2, JFS 등`이 존재



#### 3. 네트워킹

- TCP / IP 기반 인터넷에 연결하거나 응용 프로그램이 네트워크를 사용하려면,

  <u>OS에서 네트워크 프로토콜을 지원해야 함</u>



#### 4. 사용자 관리

- 한 컴퓨터를 여러 사람이 사용하는 환경도 지원해야 함

  => OS는 각 계정을 관리할 수 있는 기능이 필요.

  ​     ex) 사용자 별 프라이버시 보장, 파일에 대한 접근 권한 부여 등



#### 5. 디바이스 드라이버

- OS는 시스템의 자원, 하드웨어를 관리하는데, 

  이들을 OS에서 인식하고 관리하게 하여 <u>응용 프로그램이 하드웨어를 사용할 수 있도록 해야 함</u>

  => 따라서 OS 내부에 ***하드웨어를 추상화 해주는 계층이 필요***

  ​									=> 이 계층이 디바이스 드라이버

- OS는 수많은 디바이스 드라이버를 관리함
